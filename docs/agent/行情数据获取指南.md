# A 股行情数据获取指南

## 概述

FreshQuant 使用 QUANTAXIS 作为数据源，配合 MongoDB 缓存和实时数据补充，提供 A 股日线/分钟线数据获取能力。

## 核心函数

### 1. 获取交易日历

**函数**：`fq_trading_fetch_trade_dates`

**来源**：`freshquant.trading.dt`

**用途**：获取 A 股交易日历（用于确定数据范围、计算上一交易日等）

```python
from freshquant.trading.dt import fq_trading_fetch_trade_dates

trade_dates = fq_trading_fetch_trade_dates()
# 返回：DataFrame，列名 'trade_date'，值为 date 对象
```

**典型用法**：

```python
# 获取最近 5000 个交易日
trade_dates = fq_trading_fetch_trade_dates()
trade_dates = trade_dates[trade_dates['trade_date'] <= datetime.now().date()]
trade_dates = trade_dates["trade_date"].tail(5000)

# 计算起止日期
start = datetime.combine(trade_dates.iloc[0], datetime.min.time())
end = datetime.combine(trade_dates.iloc[-1], datetime.min.time())
```

**辅助函数**：

| 函数 | 说明 |
|------|------|
| `query_current_trade_date()` | 获取当前或下一个交易日 |
| `query_prev_trade_date()` | 获取上一个交易日 |

### 2. 获取股票列表

**函数**：`fq_inst_fetch_stock_list`

**来源**：`freshquant.instrument.stock`

**用途**：获取 A 股股票基本信息（代码、名称、交易市场等）

```python
from freshquant.instrument.stock import fq_inst_fetch_stock_list

stock_list = fq_inst_fetch_stock_list()
# 返回：List[Dict]，每个字典包含 code, name, volunit, decimal_point, sse, sec 等字段
```

**典型用法**：

```python
# 获取所有股票
stock_list = fq_inst_fetch_stock_list()

# 过滤 ST 股
stock_list = [s for s in stock_list if "ST" not in s.get("name", "")]

# 筛选指定股票
stock_list = [s for s in stock_list if s.get("code") == "000001"]
```

**返回字段说明**：

| 字段 | 说明 |
|------|------|
| code | 股票代码 |
| name | 股票名称 |
| sse | 交易所 (sh/sz) |
| sec | 板本类型 |
| volunit | 交易单位 |
| decimal_point | 小数位数 |

### 3. 获取日线数据

**函数**：`fq_data_stock_fetch_day`

**来源**：`freshquant.data.stock`

**用途**：获取单只股票的日线 OHLCV 数据

```python
from freshquant.data.stock import fq_data_stock_fetch_day

data = fq_data_stock_fetch_day(
    code="000001",           # 股票代码
    start=datetime(2023, 1, 1),  # 起始日期（可选）
    end=datetime.now()       # 结束日期（可选）
)
# 返回：DataFrame 或 None，索引为 datetime
```

**返回 DataFrame 结构**：

| 列名 | 说明 |
|------|------|
| datetime | 日期时间（索引） |
| open | 开盘价 |
| high | 最高价 |
| low | 最低价 |
| close | 收盘价 |
| volume | 成交量 |
| amount | 成交额 |
| time_stamp | 时间戳 |

**完整示例**：

```python
from datetime import datetime, timedelta
from freshquant.trading.dt import fq_trading_fetch_trade_dates
from freshquant.data.stock import fq_data_stock_fetch_day

async def fetch_stock_day_data(code: str, dt: datetime = datetime.now()) -> pd.DataFrame:
    """获取指定股票到目标日期的日线数据"""
    try:
        # 获取交易日历
        trade_dates = fq_trading_fetch_trade_dates()
        trade_dates = trade_dates[trade_dates['trade_date'] <= dt.date()]
        trade_dates = trade_dates["trade_date"].tail(5000)

        # 计算起止日期
        start = datetime.combine(trade_dates.iloc[0], datetime.min.time())
        end = datetime.combine(trade_dates.iloc[-1], datetime.min.time())

        # 获取数据
        return fq_data_stock_fetch_day(code=code, start=start, end=end)
    except Exception as e:
        logger.info(f"Error fetching data for stock {code}: {e}")
        return None
```

### 4. 获取分钟线数据

**函数**：`fq_data_stock_fetch_min`

**来源**：`freshquant.data.stock`

```python
from freshquant.data.stock import fq_data_stock_fetch_min

data = fq_data_stock_fetch_min(
    code="000001",
    frequence="1min",  # 1min, 5min, 15min, 30min, 60min
    start=datetime.now() - timedelta(days=7),
    end=datetime.now()
)
```

**支持频率**：`1min`, `5min`, `15min`, `30min`, `60min`

## 数据源与缓存

### 数据流向

```
akshare/QUANTAXIS → MongoDB (stock_day/stock_min) → Redis 缓存 → 内存缓存
                                      ↓
                              MongoDB (stock_realtime) ← 实时数据补充
```

### 缓存策略

| 函数 | 缓存类型 | 过期时间 |
|------|----------|----------|
| `fq_trading_fetch_trade_dates` | Redis | 86400s (24h) |
| `fq_inst_fetch_stock_list` | 内存 | 3600s (1h) |
| `fq_data_QA_fetch_stock_day_adv` | Redis | 900s (15min) |
| `fq_data_QA_fetch_stock_min_adv` | Redis | 900s (15min) |

### 数据补充

日线/分钟线数据会自动从 `stock_realtime` 集合补充 QUANTAX 之后的实时数据。

## 股票代码规范

| 格式 | 说明 |
|------|------|
| `000001` | 纯代码（默认深市） |
| `600000` | 纯代码（默认沪市） |
| `000001.sz` | 代码.市场 |
| `sz000001` | 市场+代码 |

**注意**：内部会自动处理市场代码，使用 `fq_util_code_append_market_code` 转换。

## 注意事项

1. **数据长度**：技术分析通常需要足够的历史数据（建议 5000+ 交易日）
2. **前复权**：日线数据自动前复权处理 (`to_qfq()`)
3. **异常处理**：数据获取可能失败，需要 try-except 包裹
4. **时间过滤**：使用交易日历过滤非交易日数据
5. **Redis 缓存**：频繁调用可利用缓存减少重复查询
