# 项目目标与代码现状调研

- **更新日期**：2026-03-10

## 1. 项目目标与治理基线

本仓库不是旧仓 `D:\fqpack\freshquant` 的镜像，而是 **迁移后的目标架构**。当前治理规则来自根 `AGENTS.md`、`docs/migration/README.md` 与 CI 门禁：

- 在“重构与去冗余”的前提下，逐步迁移旧仓功能。
- 允许破坏性变更，但以下事项必须先写 RFC 并评审通过后才能编码：
  - 新增顶级模块
  - 新增 CLI/API/worker/service/scheduler 入口
  - 跨多个目录的大重构
  - 新增外部依赖
  - 任何破坏性变更
- 迁移治理采用文件化记录：
  - `docs/rfcs/`
  - `docs/migration/progress.md`
  - `docs/migration/breaking-changes.md`
- Git 规则是：
  - 禁止直推 `main`
  - 禁止在本地 `main` 直接开发
  - 必须使用 `git worktree + feature branch`
  - 合并路径固定为 `feature -> PR -> origin/main -> 本地 main fast-forward`

## 5. 配置与运行环境

### 5.1 Python 与依赖

- 当前统一运行面：`Python 3.12.x`
- 依赖锁定：`uv.lock`
- 主 CLI/测试推荐用法：`uv run ...`
- Docker 中分别构建：
  - `/freshquant/.venv`
  - `/app/.venv`（TradingAgents-CN）

### 5.2 FreshQuant 配置来源

FreshQuant 仍是多来源配置：

- `freshquant/config.py` 的 Dynaconf
- 仓库根 `.env`（Docker Compose 注入）
- Mongo `freshquant.params` 运行时参数

当前需要特别注意的配置层：

- `order_management.*`
- `position_management.*`
- `mongodb.gantt_db`
- `xtquant.account_type / account / path`

### 5.3 TradingAgents-CN 配置来源

TradingAgents-CN 当前已按 RFC `0016` 收敛为：

- 仓库根 `.env` 是单一真相源
- `docker/compose.parallel.yaml` 通过 `env_file: ../.env` 注入
- 启动阶段同步到 Mongo 配置镜像
- 运行期 `config_bridge` 采用环境变量优先

不应再把 `third_party/tradingagents-cn/.env` 视为正式配置入口。

## 8. 当前代码现状观察

### 8.1 已经收敛的部分

- ETF 与股票查询默认复权语义已对齐
- XTData 实时链路已落地 producer/consumer/fullcalc
- 订单、仓位、止盈止损已经拆成独立域模块
- Gantt / Shouban30 已有独立读模型、API 和盘后快照链路
- Gantt 统一页面已收敛到共享 `GanttHistory` 图表层，并补齐 legacy 连板语义配色、legend、hover 和 viewport 同步
- KlineSlim 已收敛为“默认 5m + 图例按需打开 1m/15m/30m”的多周期缠论显示语义
- 运行观测已收敛到 `runtime_observability` 公共包，并提供 `/api/runtime/*` 与 `/runtime-observability`
- TradingAgents-CN 已纳入并行部署和配置治理

### 8.2 仍需持续注意的部分

- 运行框架并存：
  - Tornado
  - Flask
  - Dagster
  - Huey
  - 宿主机 XT broker
  - 独立 FastAPI（TradingAgents-CN）
- 配置来源仍有分层复杂度：
  - Dynaconf
  - Mongo `params`
  - Docker `.env`
  - TradingAgents 启动期 Mongo 镜像配置
- Web 前端存在“源码 + 构建产物”双份事实：
  - `morningglory/fqwebui/src/**`
  - `morningglory/fqwebui/web/**`
  - 涉及页面行为调整时，必须同步确认构建产物已更新
- 运行观测虽然接入了主链路，但必须始终保持“旁路、异步、可丢弃”，不允许阻塞策略、订单和执行路径
- 旧仓调研文档中涉及的某些模块并未在目标仓保持同名实现，例如旧的结构化日志与 `SystemLogs.vue` 体系不应直接当作当前代码事实引用
- 持仓读取当前应优先理解为“订单域投影 + `xt_positions` 补充”，而不是单纯依赖 legacy `stock_fills`
