# 配置管理指南

## 概述

FreshQuant 使用 [dynaconf](https://www.dynaconf.com/) 作为配置管理工具，支持多层级配置（环境变量 > 配置文件 > 默认值）。

## 配置文件位置

配置文件 `freshquant.yaml` 的加载优先级（从高到低）：

1. 当前工作目录 `./freshquant.yaml`
2. 可执行文件目录 `<exe_dir>/freshquant.yaml`
3. 用户主目录 `~/.freshquant/freshquant.yaml`
4. 包安装目录 `<package_dir>/freshquant.yaml`

## 环境变量规范

### 命名规则

环境变量格式：`freshquant_<SECTION>__<KEY>`

- 使用 `freshquant` 前缀
- 节点（SECTION）与键（KEY）之间用双下划线 `__` 分隔
- 全大写

### API 配置

```yaml
# freshquant.yaml
api:
  base_url: http://host.docker.internal
```

环境变量覆盖：

```bash
# Linux/macOS
export freshquant_API__BASE_URL="http://192.168.1.100:8080"

# Windows PowerShell
$env:freshquant_API__BASE_URL="http://192.168.1.100:8080"

# Windows CMD
set freshquant_API__BASE_URL=http://192.168.1.100:8080
```

### 其他常用配置

```yaml
# MongoDB 配置
mongodb:
  host: localhost
  port: 27017
  db: freshquant
  gantt_db: freshquant_gantt

order_management:
  mongo_database: freshquant_order_management
  projection_database: freshquant
  external_confirm_seconds: 120
  guardian_allocation_policy: guardian_compat_low_price_first

# Redis 配置
redis:
  host: 127.0.0.1
  port: 6379
  db: 1

# 代理配置
proxy:
  host: 127.0.0.1
  port: 10809
```

对应环境变量：

```bash
export freshquant_MONGODB__HOST="mongo.example.com"
export FRESHQUANT_MONGODB__GANTT_DB="freshquant_gantt_prod"
export FRESHQUANT_ORDER_MANAGEMENT__MONGO_DATABASE="freshquant_order_management_prod"
export FRESHQUANT_ORDER_MANAGEMENT__PROJECTION_DATABASE="freshquant"
export freshquant_REDIS__HOST="redis.example.com"
export freshquant_PROXY__HOST="127.0.0.1"
```

`mongodb.gantt_db` 用于 Gantt / Shouban30 专题域的独立分库，默认值为 `freshquant_gantt`。该分库用于保存 XGB / JYGS 原始同步数据与页面读模型，避免与 `freshquant` 主库中的交易、参数、持仓等数据混放。

`order_management.mongo_database` 用于订单管理主账本独立分库，默认值为 `freshquant_order_management`；`order_management.projection_database` 默认为 `freshquant`，用于兼容投影（如 `stock_fills`）继续落在现有业务库。

## 代码中使用配置

### API 地址（推荐公共函数）

```python
from freshquant.util.url import get_api_base_url

# 获取 API 基础地址
base_url = get_api_base_url()
url = f'{base_url}/api/endpoint'
```

### 其他配置（标准用法）

```python
from freshquant.config import settings

# 获取 Redis 配置
redis_host = settings.get('redis', {}).get('host', '127.0.0.1')
redis_port = settings.get('redis', {}).get('port', 6379)

# 获取 MongoDB 配置
mongo_url = settings.get('mongodb', {}).get('url', 'mongodb://localhost:27017')
```

## Docker 环境配置

### API 服务地址

在 Docker 容器中访问宿主机服务时，使用 `http://host.docker.internal`：

```yaml
# freshquant.yaml
api:
  base_url: http://host.docker.internal
```

这是默认配置，无需修改。如需指向其他服务器，通过环境变量覆盖：

```bash
docker run -e freshquant_API__BASE_URL="http://api.example.com" ...
```

## 配置优先级示例

```python
# 配置文件
api:
  base_url: http://host.docker.internal  # 默认值

# 环境变量
freshquant_API__BASE_URL=http://192.168.1.100:8080  # 最高优先级

# 代码中获取
from freshquant.util.url import get_api_base_url
base_url = get_api_base_url()
# 结果: http://192.168.1.100:8080
```

## 注意事项

1. **环境变量优先**：生产环境应使用环境变量覆盖敏感配置
2. **类型转换**：dynaconf 会自动处理类型（字符串、数字、布尔等）
3. **多配置文件**：支持 `freshquant_*.yaml` 格式的扩展配置
4. **验证**：获取配置后建议验证有效性，特别是网络地址

## Windows PowerShell 中文显示（UTF-8）

在 Windows PowerShell 5.1 中，`cat/type/Get-Content` 默认按 ANSI（gb2312）读取“无 BOM 的 UTF-8 文件”，会导致中文乱码。本仓库文档/配置文件默认是 UTF-8（无 BOM）。

**解决方案（二选一）**：

1. **临时（推荐）**：进入仓库根目录后执行一次（需要 dot-source）：

```powershell
. .\script\pwsh_utf8.ps1
```

2. **单次读取**：对个别文件显式指定编码：

```powershell
Get-Content -Encoding utf8 docs\agent\index.md
```

**验证**：执行 `cat docs\agent\index.md` 应能正常显示中文。

## Docker 并行 + 宿主机运行时

当使用 `docker/compose.parallel.yaml` 启动本仓库时，Docker Mongo 暴露在宿主机 `127.0.0.1:27027`。
如果 `broker`、`freshquant.market_data.xtdata.market_producer`、`freshquant.market_data.xtdata.strategy_consumer` 仍运行在 Windows 宿主机，则必须显式覆盖 Mongo 端口，否则它们会回落到默认 `27017`，与 Docker 内 API 读写不同的库。

推荐环境变量：

```bash
export FRESHQUANT_MONGODB__HOST="127.0.0.1"
export FRESHQUANT_MONGODB__PORT="27027"
export FRESHQUANT_REDIS__HOST="127.0.0.1"
export FRESHQUANT_REDIS__PORT="6379"
```

Windows PowerShell：

```powershell
$env:FRESHQUANT_MONGODB__HOST="127.0.0.1"
$env:FRESHQUANT_MONGODB__PORT="27027"
$env:FRESHQUANT_REDIS__HOST="127.0.0.1"
$env:FRESHQUANT_REDIS__PORT="6379"
```

注意：

- MiniQMT / XTData 仍要求运行在 Windows 宿主机，不在 Docker 容器内。
- Docker `freshquant` 库首次使用前，先执行 `python -m freshquant.initialize --quiet`。
- 若宿主机旧库中已有有效 `freshquant.params`，应先同步到 Docker `freshquant.params`，再重启宿主机 `broker / producer / consumer`。
