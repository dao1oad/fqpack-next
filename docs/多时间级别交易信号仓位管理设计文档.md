## 1. 设计概述

本方案旨在根据日线、30分钟线和5分钟线三个时间级别的买卖信号，通过加权综合评估的方式确定合理的仓位水平。系统根据六个状态信号（日线买、日线卖、30F买、30F卖、5F买、5F卖）的组合，动态计算仓位配置比例。

## 2. 核心设计原则

### 2.1 时间级别权重分配

- **日线级别**：权重最高（3），作为主要趋势判断依据
- **30分钟级别**：权重中等（2），作为中期趋势参考
- **5分钟级别**：权重最低（1），作为短期波动参考

### 2.2 边界条件

- 全部卖出信号时：仓位 = 20%
- 全部买入信号时：仓位 = 80%

### 2.3 映射关系

采用线性映射方式，确保仓位变化平滑且可预测。

## 3. 技术实现方案

### 3.1 状态编码规则

| 时间级别 | 买入信号 | 卖出信号 |
| ---- | ---- | ---- |
| 日线   | +1   | -1   |
| 30分钟 | +1   | -1   |
| 5分钟  | +1   | -1   |

### 3.2 权重配置

| 时间级别 | 权重系数 |
| ---- | ---- |
| 日线   | 3    |
| 30分钟 | 2    |
| 5分钟  | 1    |

### 3.3 计算公式

#### 综合得分计算：

```
综合得分 = (日线信号 × 3) + (30分钟信号 × 2) + (5分钟信号 × 1)
```

#### 仓位映射公式：

```
仓位比例 = 20% + (综合得分 + 6) × 5%
```

#### 公式推导：

- 得分范围：[-6, 6]
- 仓位范围：[20%, 80%]
- 每单位得分对应仓位变化：60% / 12 = 5%

### 3.4 计算示例

| 场景  | 日线    | 30分钟  | 5分钟   | 综合得分 | 仓位计算            | 最终仓位 |
| --- | ----- | ----- | ----- | ---- | --------------- | ---- |
| 全卖出 | 卖(-3) | 卖(-2) | 卖(-1) | -6   | 20% + (-6+6)×5% | 20%  |
| 全买入 | 买(+3) | 买(+2) | 买(+1) | +6   | 20% + (6+6)×5%  | 80%  |
| 混合1 | 买(+3) | 卖(-2) | 买(+1) | +2   | 20% + (2+6)×5%  | 60%  |
| 混合2 | 卖(-3) | 买(+2) | 卖(-1) | -2   | 20% + (-2+6)×5% | 40%  |

## 4. 算法伪代码

```python
def calculate_position(daily_signal, thirty_min_signal, five_min_signal):
    """
    根据三个时间级别的信号计算仓位比例

    参数:
    daily_signal: str, "buy" 或 "sell"
    thirty_min_signal: str, "buy" 或 "sell"
    five_min_signal: str, "buy" 或 "sell"

    返回:
    float: 仓位比例(20-80之间的数值)
    """

    # 信号数值转换
    daily_val = 1 if daily_signal == "buy" else -1
    thirty_val = 1 if thirty_min_signal == "buy" else -1
    five_val = 1 if five_min_signal == "buy" else -1

    # 权重配置
    w_daily = 3
    w_thirty = 2
    w_five = 1

    # 计算综合得分
    total_score = (daily_val * w_daily +
                  thirty_val * w_thirty +
                  five_val * w_five)

    # 映射到仓位比例
    position = 20 + (total_score + 6) * 5

    return position
```

## 5. 特性分析

### 5.1 优势

- **趋势主导**：日线权重最高，确保主要趋势方向的主导地位
- **平滑过渡**：线性映射保证仓位变化的连续性
- **灵活可调**：权重系数可根据实际需求调整
- **边界明确**：严格满足20%-80%的仓位边界要求

### 5.2 扩展性

- 可增加更多时间级别信号
- 支持非线性映射函数替换
- 权重系数可动态调整

## 6. 使用建议

1. **信号质量**：确保输入信号的准确性，建议使用经过验证的技术指标
2. **风险控制**：结合止损策略使用，避免单一信号失效风险
3. **回测验证**：在实际应用前进行充分的历史回测
4. **参数优化**：可根据具体交易品种调整权重参数
