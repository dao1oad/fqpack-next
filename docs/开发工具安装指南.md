# 开发工具安装指南

> 当前项目运行环境已收敛为 `Python 3.12 + uv + 项目根目录 .venv`。本文只补充 Windows 本机构建某些依赖时可能需要的编译工具，不再代表完整环境安装流程。

## 当前环境基线

- Python 依赖安装统一使用 `uv sync --frozen`
- 日常命令统一使用 `uv run ...`
- FreshQuant 宿主机默认虚拟环境为项目根目录 `.venv`
- TradingAgents-CN 在 Docker 并行模式下使用容器内独立 `.venv`

如需完整项目入口、部署和配置口径，优先参考：

- `docs/agent/项目目标与代码现状调研.md`
- `docs/agent/Docker并行部署指南.md`
- `docs/agent/配置管理指南.md`
- `docs/agent/Symphony本地安装与使用指南.md`（本机评估 OpenAI `Symphony` 时）

## 安装C++编译工具

从下面地址下载和安装编译工具：

https://visualstudio.microsoft.com/visual-cpp-build-tools/

安装的只是编译工具，并不安装Visual Studio集成开发环境，这样占用的磁盘空间小一点。想安装完整的集成开发环境也可以选择安装Visual Studio 2022 Community。

https://visualstudio.microsoft.com/zh-hans/free-developer-offers/

安装其中一个就行。

## 何时需要安装

以下场景才通常需要额外安装 C++ Build Tools：

- 本机编译含原生扩展的 Python 依赖
- 本机重建某些历史二进制包或开发调试扩展模块
- 遇到 `Microsoft Visual C++ 14.x` 缺失类错误

如果 `uv sync --frozen` 已能直接完成依赖安装，则不需要额外处理本文件中的编译工具步骤。
