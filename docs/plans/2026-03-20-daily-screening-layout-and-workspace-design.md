# 每日选股布局与工作区交互优化设计

## 背景

`/daily-screening` 已经切到“Dagster 预计算 + 前端交集查询”的工作台，但当前页面仍有四个明显问题：

- 工作区表格选中标的后，右侧没有像交集列表一样同步展示完整标的详情
- 顶部“工作台说明”占高，挤压主工作区
- 页面在 100% 缩放下会因为固定高度和全页裁切导致右侧详情底部内容不可见
- 交集列表缺少单条加入 `pre_pools` 的入口，左侧基础池条件也没有被单独归组

## 目标

- 工作区点击行时，右侧复用交集列表同一套详情接口，展示全部详情卡片
- 顶部说明改成单行并排标签，减少纵向占用
- 页面在 100% 缩放下也能完整使用，不再依赖 90% 缩放
- 右侧卡片整体更紧凑，把更多高度留给“历史热门理由”
- 交集列表每行增加“加入 pre_pools”按钮，单标的追加去重
- `CLS 模型分组` 和 `热门窗口` 作为“基础池（并集）”独立分组展示
- `日线缠论涨幅` 默认启用参与筛选

## 非目标

- 不为每日选股单独新建一套工作区存储
- 不改动 Dagster 每日选股正式结果口径
- 不把页面重新做回“页面内运行器”或恢复 SSE

## 已确认口径

- 单标的加入 `pre_pools` 继续复用 `/gantt/shouban30` 的共享工作区追加接口
- 工作区和交集列表都只读写同一套共享 `pre_pools / stock_pools / must_pools`
- 工作区点击行后，右侧详情调用 `/api/daily-screening/stocks/<code>/detail`
- “日线缠论涨幅”在交集列表已有展示后，右侧单独卡片可以移除

## 根因分析

100% 缩放显示不全不是单一 CSS 细节，而是三层约束叠加：

- 全局 `workbench-page` / `workbench-body` 使用 `height: 100vh` + `overflow: hidden`
- 每日选股页面中间和右侧多张表格写死 `height="380" / "260" / "280"`
- 右侧详情区是纵向堆叠的 4 张卡片，没有可伸缩分配策略

结果是页面高度一旦不够，底部卡片既不能自然压缩，也不能由页面整体滚动承载，只会被容器直接裁掉。

## 方案比较

### 方案 1：继续堆叠右侧卡片，只把单个表格高度调小

优点：

- 改动少

缺点：

- 只是继续调数字
- 不解决固定高度 + 全页裁切的结构问题
- 后续再加内容还会复发

### 方案 2：右侧改成“摘要 + 紧凑条件卡片网格 + 历史热门理由主卡”，并把中间区改为弹性高度

优点：

- 直接消除固定高度导致的裁切
- 信息层级更清晰
- 右侧能把主要高度留给“历史热门理由”

缺点：

- 需要同时改模板、状态和 CSS

## 结论

采用方案 2。

页面会把“说明”“基础池条件”“结果表格”“工作区”“详情区”都收敛成弹性布局：

- 顶部说明改为标签行
- 左侧把 `CLS 模型分组` 和 `热门窗口` 放入“基础池（并集）”分组
- 中间区保留交集列表和工作区，但去掉硬编码表格高度
- 右侧拆成两个层次：
  - 顶部摘要与紧凑条件卡片
  - 底部“历史热门理由”主卡占据剩余高度

## 详细设计

### 1. 工作台说明

把 `workbenchGuideLines` 从无序列表改成标签数组，直接在 toolbar 里一行换行展示：

- 保留原有四条业务语义
- 改成 chip/tag 视觉，不再占据整块卡片高度

### 2. 左侧筛选工作台

`conditionSections` 调整为两组：

- `basePool`
  - `CLS 模型分组`
  - `热门窗口`
- `intersection`
  - `市场属性`
  - `chanlun 周期`
  - `chanlun 信号`

这样视觉上明确“基础池先并集、其他条件再交集”。

同时：

- `buildDailyScreeningWorkbenchState()` 默认把 `dayChanlunEnabled` 设为 `true`
- scope 切换重置时也保持默认启用，而不是回到关闭状态

### 3. 交集列表

每行在名称后增加“操作”列：

- 按钮文案：`加入 pre_pools`
- 动作：把当前行转成单条共享工作区 append payload，再调用 `appendShouban30PrePool`
- 去重：依赖共享工作区 append 接口的 `skipped_count` 语义

### 4. 工作区行点击详情

工作区表格增加 `@row-click`：

- 读取点击行 `code6`
- 同步更新 `selectedCode`
- 调用与交集列表完全相同的 `loadDetail(code)`

这样工作区选中标的也能展示：

- 标的摘要
- CLS/热门窗口/市场属性/chanlun 周期/chanlun 信号
- 历史热门理由

### 5. 右侧详情区

右侧不再使用“摘要 / 命中条件 / 日线缠论涨幅 / 历史热门理由”四张纵向卡片，而是改成：

- 摘要卡
- 条件卡片区
  - `CLS 模型`
  - `热门窗口`
  - `市场属性`
  - `chanlun 周期`
  - `chanlun 信号`
  - `缠论原因`
- 历史热门理由主卡

其中：

- “日线缠论涨幅”独立卡片删除
- 数值信息保留在交集列表和摘要区中的紧凑说明里
- “历史热门理由”使用剩余高度并保持表格滚动

### 6. 布局修正

关键原则：

- 去掉每日选股页面内部硬编码表格高度
- 每个大区块用 `display: flex` / `grid` + `min-height: 0`
- 只让真正需要滚动的表格区域承担滚动

具体会做：

- 中间 stack 和右侧 stack 设为 `minmax(0, 1fr)` 体系
- 交集列表、工作区、历史热门理由分别放在 `flex` 子区块中
- 通过 `workbench-table-wrap` 包裹表格，让表格吃剩余空间
- 右侧详情条件卡区改成 2 列紧凑网格，窄屏时自动回到 1 列
