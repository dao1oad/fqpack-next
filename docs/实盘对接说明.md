# 实盘对接说明

目前对接实盘只接入MiniQMT的方式，由MiniQMT提供下单接口。MiniQMT是运行在本地的一个客户端，然后你可以使用程序连接上去进行下单，MiniQMT上也有提供行情的服务。

## 实盘对接有哪几部分组成

实盘运行由一下几个组件协作完成

1. 在Docker中运行的fq_guardian，他的程序入口是：
```
python -m freshquant.signal.astock.job.monitor_stock_zh_a_min
```
这个程序是监控股票生成信号数据，并且把买入卖出信号发送到redis队列

2. 需要在MiniQMT同一台电脑上运行的fq_xtrade，他的程序入口是：
```
python -m fxqtrade.xtquant.broker
```
他是兼听redis队列，接收到订单消息后，下单到MiniQMT客户端，实现了实盘下单的功能

3. 相关的参数配置在数据库中的params表中，配置的顶级key是xtquant和monitor两个。

## Docker 并行模式补充

当本仓库使用 `docker/compose.parallel.yaml` 并行部署时：

- Docker 内 API / Dagster 读取的是 `fq_mongodb`，宿主机映射端口为 `27027`
- `broker`、`xtdata producer`、`xtdata consumer` 仍运行在 Windows 宿主机

因此宿主机运行时必须显式配置：

```text
FRESHQUANT_MONGODB__HOST=127.0.0.1
FRESHQUANT_MONGODB__PORT=27027
```

否则会回落到默认 `27017`，造成 Docker 内服务与宿主机 MiniQMT 链路读写两套 Mongo 数据。

首次切换到 Docker Mongo 时，先执行：

```powershell
$env:FRESHQUANT_MONGODB__HOST="127.0.0.1"
$env:FRESHQUANT_MONGODB__PORT="27027"
python -m freshquant.initialize --quiet
```

如果宿主机旧库中已经存在 `freshquant.params`，应先同步到 Docker 的 `freshquant.params`，再重启宿主机 `broker / producer / consumer`。
